# Principles

## Evidence Before Claims

The system must write evidence records before it writes conclusions. A report is
only a readable view of the underlying evidence ledger.

## No Automatic Misconduct Verdicts

The tool reports risk signals and verification needs. It does not determine
fabrication, falsification, plagiarism, intent, or responsibility.

## Traceability

Each finding must point to at least one source: input file, DOI metadata, page,
figure, table, data row, policy anchor, or case-card rule.

## False-Positive Awareness

Every detector spec must document false-positive risks and manual-verification
requirements before implementation.

## Local First

The default workflow runs on local files and writes local outputs. Network calls
must be explicit, limited, and documented.
