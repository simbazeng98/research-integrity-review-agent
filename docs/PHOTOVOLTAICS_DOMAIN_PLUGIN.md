# Photovoltaics & Materials Domain Plugin (v0.10)

This domain-specific plugin extends the research integrity review capabilities to photovoltaics (PV), perovskite, and materials characterization datasets.

## 1. Scope
The plugin implements consistency checks and reporting completeness checklists for solar cell datasets (single-junction, tandem, stability, and characterization summaries).

## 2. What v0.10 Checks
- **PCE Self-Consistency**: Validates that reported Power Conversion Efficiency (PCE) matches the product of Voc, Jsc, and FF, adjusted for light intensity.
- **EQE-JV current-density comparison**: Compares J-V current density with integrated EQE-derived current density.
- **Voc Loss / Bandgap Physical Limits**: Flags negative/near-zero Voc loss (< 0.05 eV) and extremely high Voc loss (> 0.65 eV).
- **Reporting Completeness Checklist**: Identifies missing parameters (cell area, scan rate/direction, statistics, light calibration).
- **Stability stress-test checklists**: Detects missing encapsulation, temperature, humidity, load/MPP tracking conditions for aging tests.
- **Tandem PV Consistency**: Validates 2T current matching and 4T efficiency summation.
- **Materials Characterization Metadata**: Flags missing radiation source (XRD), calibration reference (XPS), excitation wavelength (PL), or scale bar (SEM) parameters.

## 3. What v0.10 Does Not Check (Out of Scope)
- Does not extract tables from raw PDFs.
- Does not run OCR on J-V plots or spectra.
- Does not integrate raw EQE spectrum data (v0.11 candidate).
- Does not calculate hysteresis index from raw J-V curves (v0.11 candidate).
- Does not determine research misconduct or data fabrication.

## 4. Required Inputs
- A table package manifest (`table_manifest.jsonl`) generated by `table-intake`.
- Optional column profiles (`column_profiles.jsonl`) from table intake.

## 5. PV Field Mapping
Maps headers using regex matching (case-insensitive) to identify:
- **Voc**: `Voc`, `open-circuit voltage`, `voltage_oc`, etc.
- **Jsc**: `Jsc`, `short-circuit current density`, `mA/cm2`, etc.
- **FF**: `FF`, `fill factor`, etc.
- **PCE**: `PCE`, `efficiency`, `eta`, etc.
- **EQE Jsc**: `integrated Jsc`, `Jsc from EQE`, `IPCE Jsc`, etc.
- **Area**: `active area`, `aperture area`, `mask area`, `device area`.
- **Scan**: `scan direction`, `scan rate`.

## 6. Unit Normalization
- **Voc**: mV converted to V. Value > 20 V flags warning if unit is missing.
- **Jsc**: A/m² or A/cm² converted to mA/cm².
- **FF**: Fraction (0–1) kept; percentage (1–100) divided by 100 to get fraction. Value > 100 flags warning.
- **PCE**: Fraction (0–1) converted to percentage. Value > 100 flags warning.
- **Area**: mm² converted to cm².

## 7. PCE Consistency Check
- Formula: `PCE = (Voc * Jsc * FF / Pin) * 100%`
- Default `Pin` = 100 mW/cm² (1 sun).
- Triggers finding if difference exceeds absolute 0.3 percentage points or relative 3%.

## 8. EQE/J–V Jsc Consistency Check
- Computes `abs_diff = |JV Jsc - EQE Jsc|` and `rel_diff = abs_diff / JV Jsc`.
- Triggers medium finding if `abs_diff > 1.0 mA/cm²` and `rel_diff > 10%`.
- Triggers low note if `rel_diff > 5%` but under the medium threshold.
- Skips if `rel_diff <= 5%` (typical discrepancy).

## 9. Voc Loss / Bandgap Check
- Formula: `Voc_loss = Eg - Voc`
- Large Voc: Voc > 10 V flags mV unit mapping warning.
- Negative/near-zero loss: Voc_loss < 0.05 eV (medium risk).
- High Voc loss: Voc_loss > 0.65 eV (low risk note).

## 10. Reporting Completeness Checks
- Checks for active area, scan rate, scan direction, temperature/humidity, statistics, and simulator calibration.
- Flags gaps as low-risk reporting checks only.

## 11. Stability Reporting Checks
- Triggered by stability-related terms.
- Scans for missing temperature, humidity, load/MPP tracking, illumination, or encapsulation conditions.

## 12. Tandem PV Checks
- **2T Tandems**: Mismatch of subcell currents > 15% flags current-limiting warning.
- **4T Tandems**: Total PCE > sum of subcells + 0.3% PCE flags consistency warning.

## 13. Materials Characterization Metadata Checks
- Triggered by instrument keywords (XRD, XPS, PL, SEM, AFM, etc.).
- Cross-checks for critical settings (e.g. Cu Ka source for XRD, C1s reference for XPS, laser for PL).

## 14. False Positive Risks
- See [PV_FALSE_POSITIVE_GUIDE.md](PV_FALSE_POSITIVE_GUIDE.md).

## 15. Manual Verification Protocol
1. Retrieve raw instrument files (.opju, .dat, .csv).
2. Cross-examine spreadsheet conversion formulas.
3. Review light source and mask geometries under microscope.
4. Verify subcell testing filter configurations.

## 16. Safe Report Language
All findings use safe, non-verdict language like "Candidate consistency signal / requires verification" or "completeness gap". Do not use "fraud", "fabricated data", or "fabrication".
